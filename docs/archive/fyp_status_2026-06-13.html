The research question

When a 4-bit model looks safer, is it really safer?

Quantization shrinks a model so it can run on cheaper hardware. If its harmful-compliance rate moves afterwards, that could mean its alignment shifted, or simply that it got less capable and started refusing things it no longer understands.

Those two stories look identical on a safety benchmark alone. Telling them apart is the question this project set out to answer, and it needs a capability measurement sitting right next to the safety one.

Design: matched pairs from identical weights · on-the-fly NF4 4-bit · greedy decoding · seed 42
How the study is built

A matched-pair design

Both members of a pair load from the same weights. The only thing that changes is whether quantization is applied at load time, so any difference is attributable to quantization and nothing else.

The headline finding

One change, three different stories

4-bit quantization did not affect the three models the same way. The damage depends on model size and family, and it surfaces on different axes.

Qwen3-1.7Bsmallest +5.5 pp ✻ −2.4 pp −8.7 pp ✻ degrades on both
Qwen3-4B4 B +2.5 pp −0.8 pp −0.3 pp essentially intact

✻ = statistically significant (paired bootstrap CI excludes 0; Qwen3-1.7B ΔASR McNemar p = 0.027). Positive ΔASR = more harmful. Negative ΔMMLU = capability lost. Across all three pairs, 4-bit quantization never reduced harmful compliance.

A methodological self-correction

The framework caught its own error

My first HarmBench scorer was a deterministic refusal regex. On review it was over-counting harmful compliance, so I validated it against HarmBench's own 13B classifier model, also run on TC1.

Before

Regex proxy

Pattern-matched refusals. Fast and deterministic, but it mislabelled hedged or verbose answers as compliance and inflated the rate.

Validation

13B classifier

HarmBench's official judge model scored every response. It now serves as the primary scorer; the regex is kept only as a secondary proxy.

Effect

Finding relocated

The one significant safety regression moved from the 4B model to the 1.7B model. The framework corrected the result instead of propagating it.

Judge run on TC1 (fp16, 32 GB V100, n = 200 × 6, 0 parse errors). Outputs stored as redacted sidecars; the original raw outputs are never modified.

Stress-testing the headline

It holds, but it softens

The main study uses deterministic greedy decoding. To check the one significant safety finding (Qwen3-1.7B) was not an artifact of a single decode, I re-ran HarmBench under realistic stochastic sampling (temperature 0.7) across five independent seeds.

So the direction holds on average but the effect size roughly halves under sampling and is not consistent across every seed. The framework qualifies its own headline instead of overstating it.
